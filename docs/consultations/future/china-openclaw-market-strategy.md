# Product/roadmap changes that unlock China later

If the first wave works, then I’d consider:

## 1) APAC region before mainland region
Before doing anything mainland-compliance-heavy, add an **APAC region** (Singapore is the obvious first move).

Why:
- much better latency for China/HK/SEA
- easier than mainland infra/compliance
- helps global Chinese builders too

## 2) A clearer compliance/data-location page
Be explicit:

- where data is hosted
- what workloads are appropriate today
- what is *not* appropriate today
- when to talk to you about dedicated/private deployment

For now, market heavily to:
- prototypes
- internal tools
- non-sensitive workflows
- global apps

Be careful with:
- sensitive public-sector data
- regulated PII
- domestic production workloads in mainland China

## 3) Eventually: a partner-billed / reseller SKU
If installer demand is real, a future feature set could be:

- one partner funds a pool of allowances
- their operators create customer instances
- each instance is capped/revocable
- partner sees usage / line items / templates

That would be very strong in the China service-provider market.
